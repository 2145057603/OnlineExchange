# 卡面索引工坊

纯静态本地模组解析页面。可一次选择整个模组文件夹，自动读取 `mod_manifest.json`、PCK 和 DLL；所有读取、SHA-256 计算、JSON 提取、DLL 静态字符串分析与索引生成均在浏览器本地完成。页面不使用第三方依赖、分析服务或网络请求。

## Cloudflare Pages 部署

若仓库根目录包含本目录：

1. 在 Cloudflare Pages 连接仓库。
2. 构建命令留空。
3. 构建输出目录填写 `card-art-indexer`。
4. 部署。

若将本目录作为独立仓库根目录，构建输出目录填写 `/`。

`_headers` 已设置 CSP，禁止页面发起连接请求。

## 本地索引

输出 JSON 记录 PCK 文件名、大小、SHA-256、模组 ID/版本，以及已识别的 `cardId -> resourcePath` 映射。将输出放入 `D:\codex\OnlineExchange\indexes` 后，OnlineExchange 会在本地自动校验已加载 PCK 的 SHA-256、模组 ID 和资源路径，再导入匹配项。

## 限制

- 当前支持目录中未标记压缩的 Godot PCK 条目。
- DLL 仅提取本地静态字符串和图片资源引用，绝不会被网页执行；动态注册、反射和运行时生成的卡面不能由静态分析保证识别。
- 动态生成卡图、没有卡牌标识且无法通过路径推断的资源无法自动映射。
- 网站只生成索引，不上传或分发 PCK、图片与其他模组资源。
